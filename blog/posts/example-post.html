Building a Portfolio with Zero Dependencies

2026-03-04

This is a template blog post. Replace this content with your own writing.

Why Vanilla?

Frameworks are powerful, but for a personal portfolio with three pages, they introduce unnecessary complexity. No node_modules, no build step, no configuration files. Just HTML, CSS, and a small amount of JavaScript.

The Stack

The entire site runs on vanilla HTML, CSS custom properties for theming, and about 50 lines of JavaScript for scroll animations. It deploys directly to GitHub Pages with zero configuration.

What I Learned

Modern CSS is remarkably capable. clamp() for fluid typography, CSS Grid for layout, custom properties for design tokens — most of what frameworks provide can be achieved natively with less overhead.
